# 功能开发流程

1. 用一句话定义用户结果、影响的主题和数据兼容性。
2. 在类型、数据、编辑、预览、导出五个层面逐项判断影响；不涉及的项要明确记录原因。
3. 优先把新状态和副作用放入 `src/composables/` 或专属模块。除纯渲染编排外，不向 `resume-view.vue` 新增大量业务逻辑。
4. 为新增规则补单元测试；导入、导出和持久化至少覆盖正常、缺失字段和非法输入。
5. 执行 `bun run type-check && bun test:unit --run && bun run build`。
6. 交付时说明用户可见变化、数据兼容策略、验证命令和未覆盖的手工检查。
